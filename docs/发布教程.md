# WS Switch 发布教程

> 本文档说明如何发布新版本，包括首次配置和日常发版流程。

---

## 一、首次配置（只需做一次）

### 1. 创建 GitHub Token

1. 打开 [https://github.com/settings/tokens/new](https://github.com/settings/tokens/new)
2. Note 填写：`ws-switch release`
3. Expiration 选 `No expiration`（或按需设置）
4. 权限勾选：**Repository → Contents → Read and write**
5. 点击 **Generate token**，复制生成的 Token（格式：`ghp_xxx`）

### 2. 写入本地 .env 文件

打开 `d:\github_dir\ws-switch\.env`，写入：

```
GITHUB_TOKEN=ghp_你的token
```

> ⚠️ `.env` 已被 `.gitignore` 屏蔽，不会上传到 GitHub，请妥善保管。

---

## 二、发布新版本（日常流程）

### 第一步：修改版本号

打开 `account_manager.py`，找到第 19 行，修改 `VERSION`：

```python
VERSION = '1.3.0'   # 改成新版本号
```

> 版本号规则：`主版本.次版本.补丁`，例如 `1.2.1`（小修复）、`1.3.0`（新功能）

### 第二步：更新文档版本号

打开 `docs\使用教程.md`，将以下两处版本号同步更新：

- 第 3 行：`> 适用版本：WS Switch vX.X.X`
- 最后一行：`*WS Switch vX.X.X | 仅供个人学习使用*`

### 第三步：提交文档变更

```powershell
git add docs\使用教程.md README.md
git commit -m "docs: 更新版本号至 vX.X.X"
git -c http.sslBackend=openssl push origin main
```

### 第四步：一键发布

在项目目录下运行：

```powershell
.\release.ps1
```

脚本会自动完成：

| 步骤 | 操作 |
|------|------|
| 1/4 | 调用 `build.py` 打包 `ws-switch.exe` |
| 2/4 | 通过 GitHub API 创建 Release |
| 3/4 | 上传 `ws-switch.exe` 到 Release Assets |
| 4/4 | 推送 git tag |

完成后控制台输出：

```
[OK] https://github.com/aixiaoxin123/ws-switch/releases/tag/vX.X.X
```

---

## 三、重新发布同一版本（覆盖 exe）

如果已发布的版本需要替换 exe，直接再次运行：

```powershell
.\release.ps1
```

脚本会自动删除旧的 `ws-switch.exe` 资产，重新上传新的。

---

## 四、Token 失效时的处理

Token 过期后，重新生成并更新 `.env` 文件即可：

```
GITHUB_TOKEN=ghp_新token
```

---

## 五、文件结构说明

```
ws-switch/
├── .env                  ← GitHub Token（本地私有，不上传）
├── release.ps1           ← 发布脚本（本地私有，不上传）
├── build.py              ← 构建脚本（本地私有，不上传）
├── account_manager.py    ← 源码，VERSION 变量在此（不上传）
├── .github/
│   └── workflows/
│       └── release.yml   ← GitHub Actions（已上传，可选）
└── docs/
    ├── 使用教程.md        ← 用户文档（已上传）
    └── 发布教程.md        ← 本文档（已上传）
```
