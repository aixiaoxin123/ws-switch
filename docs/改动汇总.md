## 六、windsurf_switcher.py 改动（2026-04-08）

### 编码修复（CP936 → UTF-8）
`find_windsurf_exe()` PowerShell fallback 中 `subprocess.run` 的 `text=True` 在中文 Windows 系统下默认使用 CP936 编码，导致含中文字符的 exe 路径乱码、`os.path.isfile()` 返回 `False`，最终回落到兜底路径或返回 `None`。

**修复**：将 `text=True` 改为 `encoding='utf-8', errors='replace'`，与其他 `subprocess.run` 调用保持一致。

### 死代码清理
`get_plan_status()` 函数 `return None` 语句后遗留了一段永不执行的 `requests.post` 代码块，已删除。

### Firebase Referer 限制修复（刷新余额全部失败根因）
Firebase API Key 配置了 HTTP Referer 白名单，Python `requests` 默认不发送 `Referer` 头，导致所有 `firebase_login` 调用返回 `HTTP 403: Requests from referer <empty> are blocked`，即刷新余额/Token 全部失败。

**修复**：在 `firebase_login()` 的 `requests.post` 中加入：
```python
headers={'Referer': 'https://windsurf.com', 'Origin': 'https://windsurf.com'}
```
诊断过程验证了 `windsurf.com` / `www.windsurf.com` / `app.windsurf.com` / `login.windsurf.com` 均可通过，`codeium.com` 和 `chrome-extension://` 被拒绝。
